# Delivery Performance Analysis | Análisis de Rendimiento de Entregas

## English

### Objective
Analysis of delivery performance for Olist, a Brazilian e-commerce company, using real data from 5,000 orders to identify delay patterns and propose operational improvements.

### Tools Used
- **SQL** (SQLite) — analysis queries and segmentation
- **Power BI** — interactive visualization dashboard
- **Python** (pandas, matplotlib) — analysis and charts in Google Colab

### Key Findings
- 91.9% of orders arrive before the promised delivery date
- 6.8% arrive late with an average delay of 10.6 days
- Mondays and Tuesdays concentrate the highest number of late deliveries
- 9 PM is the critical peak hour for late orders
- November 2017 was the worst month (13.7% late) — coincides with Black Friday

### Files
- `queries.sql` — all SQL queries used in the analysis
- `delivery_analysis.ipynb` — Python notebook with charts
- `delivery_analysis.pbix` — Power BI dashboard
- `olista_data.csv` — dataset used

### Dataset
Based on public data from Olist Brazilian E-Commerce.
